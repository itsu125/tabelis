# アプリケーション名
Tabelis（タベリス）

# アプリケーション概要
行きたい・行ったお店を簡単に管理できるパーソナル飲食店リストアプリ。カテゴリやタグによる絞込検索が可能で、お店の評価やメモも記録できます。

# 利用方法
## ショップ登録
ログイン後、ヘッダーの「+」ボタンから飲食店を新規登録します。
   - 店名、メモ、住所、URL、カテゴリを入力できます。
   - シーンに応じてタグ（複数選択）ができます。
   - 評価は5段階で登録できます。

## ショップ閲覧
   - ショップ一覧では「行きたい」「行った」をタブで切り替えて表示することが可能です。
   - ショップ詳細ページでは登録した画像・評価・メモなどが一覧で表示され、編集・削除することが可能です。

## ショップ検索
ショップ一覧の検索ボタンから絞り込み検索ができます。
   - フリーワード検索（店名 / 住所 / メモなど）
   - カテゴリ検索
   - タグ検索（複数選択可）

## マイページ
「行った」「行きたい」の件数や直近の登録3件が確認できます。
プロフィール編集からユーザー名変更、アイコン画像の登録・変更が可能です。

# アプリケーションを作成した背景
テレビやSNSで情報を得て「行きたいお店」を日頃からメモしていましたが、情報が複数のアプリに散らばったり、どこに行ったか思い出せない不便さを感じていました。

特に、
- 行きたい店が埋もれて見返しにくい
- メモ・SNS・地図など記録場所がバラバラ
- 行ったお店の評価や感想を整理しづらい

といった課題がありました。

そこで、「行きたい / 行った」を分けて整理でき、必要な情報を一つのアプリで管理できる仕組みを作ろうと考えました。

Tabelis（タベリス）は、
- タブ切替での見やすい管理
- タグ複数選択やカテゴリによる検索
- 評価・メモ・画像の保存
- スマホでも使いやすいUI

など、日常的に使える“自分だけの飲食店リスト”を作れるように設計しています。
外食体験をもっと簡単に振り返り、次の店選びに活かせるアプリを目指して開発しました。

# 工夫したポイント
ユーザーが日常的に使えるアプリにするため、使いやすさと分かりやすさを最優先に設計しました。

- **行きたい / 行った をタブで簡単に切り替えられる UI**  
  一覧画面で迷わないよう、目的別にすぐアクセスできるデザインにしました。

- **検索フォームの開閉（toggle）により、画面の情報量を調整**  
  普段はすっきり、必要な時だけ詳細検索できる操作感を意識しました。

- **カラータグやカテゴリによる視覚的な分かりやすさ**  
  「友達と」「家族で」「ランチ」など、シーンごとに色を変え、パッと見で判断しやすい UI になるようにしています。

- **スマートフォンでも見やすいレイアウト**  
  飲食店を探す場面は外出先が多いため、スマホでの操作性を特に意識してデザインしました。

- **必要な情報を1画面で確認できる店舗詳細ページ**  
  画像・評価・メモ・タグ・URLをひとまとめにし、そのお店に行く判断がすぐできるように配置を工夫しました。

- **アプリ全体を通して「すぐ次の行動に移れる」構成**  
  検索 → 詳細を見る → 行った/行きたいの整理 → マイページで振り返りが途切れずに行えるよう、画面遷移と導線を設計しています。

# 改善点
- **レスポンシブデザインの最適化**  
  現状はスマートフォン向けの設計を優先していますが、PCで利用する際のレイアウト調整や表示最適化を今後進めていきたいです。

- **店舗情報の見やすさ向上**  
  お店の画像やカード表示のレイアウトを調整し、一覧ページでより直感的に情報を把握できるUIに改善する予定です。

- **タグ・カテゴリ管理機能の追加**  
  現状は初期データのみのため、ユーザー自身がタグやカテゴリを追加・編集・削除できる管理画面の実装を予定しています。

- **レビュー機能の追加**  
  行った店舗にコメントや感想を残せるレビュー機能を実装し、「行ってよかったポイント」を後から振り返りやすくしたいと考えています。

- **優先度・並び替え機能の導入**  
  行きたい店舗の中でも優先度を付けたり、評価順などに並び替えられるようなユーザー体験を向上する機能を今後追加する予定です。
